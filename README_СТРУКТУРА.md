# Структура проекта и работа с локальными файлами

## 📁 Структура папок

Проект использует следующую структуру:

```
бот+
├── bot.py                          # Основной файл бота
├── local_content_manager.py        # Менеджер локального контента
├── physics_bot.db                  # База данных SQLite
├── requirements.txt                # Зависимости
│
├── media/                          # Локальные медиафайлы (для демонстрации)
│   ├── comics/                     # Комиксы
│   ├── memes/                      # Мемы
│   ├── history/                    # Исторические справки
│   ├── movies/                     # Физика в фильмах
│   ├── calendar/                  # Календарик
│   └── notes/                      # Конспекты
│
└── scripts/                        # Вспомогательные скрипты
```

## 🚀 Быстрый старт

### 1. Создание папок

Запустите файл `create_folders.bat` или создайте папки вручную:

```
media/
├── comics/
├── memes/
├── history/
├── movies/
├── calendar/
└── notes/
```

### 2. Добавление локальных файлов

Поместите изображения в соответствующие папки:

- **Комиксы** → `media/comics/`
- **Мемы** → `media/memes/`
- **Исторические справки** → `media/history/`
- **Физика в фильмах** → `media/movies/`
- **Календарик** → `media/calendar/`
- **Конспекты** → `media/notes/`

### 3. Поддерживаемые форматы

- `.jpg` / `.jpeg`
- `.png`
- `.gif`
- `.webp`

## 🔄 Как работает система

### Приоритет отображения:

1. **База данных** - если есть записи в БД, они показываются в первую очередь
2. **Локальные файлы** - если в БД нет записей, используются файлы из папки `media/`

### Пример работы:

1. Пользователь открывает раздел "Комиксы"
2. Бот проверяет БД - есть ли комиксы?
3. Если нет → проверяет папку `media/comics/`
4. Если есть файлы → показывает их
5. Если нет ни того, ни другого → показывает сообщение "Контента пока нет"

## 📝 Добавление контента

### Через режим модератора (рекомендуется):

1. `/moderator` → введите пароль
2. Выберите раздел → "➕ Добавить"
3. Введите название
4. Отправьте фото (или пропустите)
5. Отправьте текст (или пропустите)

Контент сохраняется в базу данных.

### Через локальные файлы (для демонстрации):

1. Поместите файлы в папку `media/[категория]/`
2. Перезапустите бота
3. Файлы автоматически загрузятся при старте

**Важно:** Название файла (без расширения) будет использоваться как заголовок.

## 🔧 Технические детали

### Функции в `local_content_manager.py`:

- `scan_local_content()` - сканирует папки и загружает информацию о файлах
- `get_local_content_count(category)` - возвращает количество файлов в категории
- `get_local_content_by_index(category, index)` - получает файл по индексу
- `send_local_content_item(...)` - отправляет локальный файл пользователю

### Автоматическая загрузка:

При запуске бота автоматически:
1. Сканируются все папки в `media/`
2. Загружается информация о файлах
3. Файлы становятся доступны для показа

## ⚠️ Важные замечания

1. **Локальные файлы** используются только если в БД нет записей
2. **Название файла** (без расширения) используется как заголовок
3. **Порядок файлов** определяется алфавитным порядком имен файлов
4. **Перезапуск бота** необходим после добавления новых файлов в папки

## 📚 Примеры использования

### Добавление комикса через локальный файл:

1. Поместите `comic1.png` в `media/comics/`
2. Перезапустите бота
3. Комикс будет доступен как "comic1"

### Добавление через модератора:

1. `/moderator` → пароль
2. Комиксы → Добавить
3. Название: "Мой комикс"
4. Фото: отправьте изображение
5. Текст: "Описание комикса"

Комикс сохранится в БД и будет показываться в первую очередь.
