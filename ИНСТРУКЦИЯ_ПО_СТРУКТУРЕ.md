# Инструкция по структуре проекта и работе с локальными файлами

## 📁 Создание структуры папок

### Способ 1: Автоматически
Запустите файл `create_folders.bat` - он создаст все необходимые папки.

### Способ 2: Вручную
Создайте следующую структуру:

```
бот+
└── media/
    ├── comics/
    ├── memes/
    ├── history/
    ├── movies/
    ├── calendar/
    └── notes/
```

## 🖼️ Как добавить локальные файлы для демонстрации

### Шаг 1: Поместите файлы в папки
- Комиксы → `media/comics/`
- Мемы → `media/memes/`
- Исторические справки → `media/history/`
- Физика в фильмах → `media/movies/`
- Календарик → `media/calendar/`
- Конспекты → `media/notes/`

### Шаг 2: Поддерживаемые форматы
- `.jpg` / `.jpeg`
- `.png`
- `.gif`
- `.webp`

### Шаг 3: Перезапустите бота
После добавления файлов перезапустите бота - файлы автоматически загрузятся.

## 🔄 Как работает система

### Приоритет отображения:

1. **База данных** (приоритет 1)
   - Контент, добавленный через режим модератора
   - Показывается в первую очередь

2. **Локальные файлы** (приоритет 2)
   - Файлы из папки `media/`
   - Показываются только если в БД нет записей

### Пример:
- В БД есть 2 комикса → показываются они
- В БД нет комиксов, но есть файлы в `media/comics/` → показываются локальные файлы
- Нет ни того, ни другого → показывается "Контента пока нет"

## 📝 Добавление контента

### Через режим модератора (рекомендуется):

1. Отправьте `/moderator`
2. Введите пароль: `SimpPhysXRedactor123`
3. Выберите раздел (например, "Комиксы")
4. Нажмите "➕ Добавить"
5. **Введите название** (например, "Комикс про физику")
6. Отправьте фото (или нажмите "⏭️ Пропустить фото")
7. Отправьте текст (или нажмите "⏭️ Пропустить текст")

Контент сохраняется в базу данных и будет показываться в первую очередь.

### Через локальные файлы (для быстрой демонстрации):

1. Поместите файлы в `media/[категория]/`
2. Название файла (без расширения) будет использоваться как заголовок
3. Перезапустите бота
4. Файлы автоматически появятся в боте

**Пример:**
- Файл: `media/comics/funny_physics.png`
- В боте будет показан как: "funny_physics"

## ⚠️ Важные замечания

1. **Локальные файлы** используются только если в БД нет записей
2. **Название файла** (без расширения) используется как заголовок
3. **Порядок файлов** определяется алфавитным порядком имен
4. **Перезапуск бота** необходим после добавления новых файлов

## 🔧 Техническая информация

### Файлы проекта:

- `bot.py` - основной файл бота
- `local_content_manager.py` - менеджер локального контента
- `physics_bot.db` - база данных SQLite
- `create_folders.bat` - скрипт для создания папок

### Функции в `local_content_manager.py`:

- `scan_local_content()` - сканирует папки при запуске
- `get_local_content_count(category)` - количество файлов
- `get_local_content_by_index(category, index)` - получить файл по индексу
- `send_local_content_item(...)` - отправить файл пользователю

## 🐛 Решение проблем

### Проблема: Файлы не показываются
- Проверьте, что файлы в правильной папке
- Проверьте формат файла (должен быть изображением)
- Перезапустите бота

### Проблема: Показывается только БД, не локальные файлы
- Это нормально! БД имеет приоритет
- Удалите записи из БД или используйте локальные файлы для другой категории

### Проблема: После ввода названия бот выкидывает из режима модератора
- Убедитесь, что бот запущен с последней версией кода
- Проверьте, что обработчик текста правильно настроен
